# 微信云开发配置与部署

## 1. 前置条件

- 在微信公众平台注册小程序并取得 AppID。
- 在微信开发者工具中开通一个云开发环境。
- 将自己的微信号加入开发者或体验成员。

云环境 ID 属于环境配置，不是用户数据；仍建议写入 `.env.local` 而不是源码。AppSecret 和云 API 密钥绝不能进入本仓库或小程序包。

## 2. 客户端配置

1. 修改 `project.config.json` 的 `appid`，或在本机 `project.private.config.json` 覆盖。
2. 从 `.env.example` 创建 `.env.local`：

   ```dotenv
   TARO_APP_CLOUD_ENV=your-environment-id
   ```

3. 执行 `pnpm build:weapp`，确认待办页不再显示“本机模式”。

## 3. 创建集合

在云开发控制台创建：

| 集合 | 权限 | 用途 |
|---|---|---|
| `todos` | 仅创建者可读写 | 私人待办 |
| `articles` | 所有人可读，仅管理端可写 | 公开阅读缓存 |
| `favorites` | 仅创建者可读写 | 为后续云收藏预留；当前客户端收藏仍在本地 |

将 `cloud/rules/` 中的规则分别复制到集合的自定义安全规则。控制台若提供等价预设权限，优先选择预设，但仍需做下面的越权验证。

## 4. 部署阅读同步函数

1. 在微信开发者工具中找到 `cloudfunctions/syncReading`。
2. 选择“上传并部署：云端安装依赖”。
3. 确认 Node 运行时支持全局 `fetch`；若控制台只提供旧运行时，应先升级运行时，不要把网络请求移回客户端。
4. 在云函数控制台手动执行一次，预期返回：

   ```json
   { "ok": true, "count": 30, "fetchedAt": 0 }
   ```

   `fetchedAt` 实际为执行时间戳，条目不足时 `count` 可以小于 30。

5. 检查 `articles` 集合文档 id 形如 `hn:123456`，再次执行后文档数不应因同一批数据翻倍。
6. 检查 `config.json` 的每 6 小时定时触发器已在控制台生效；部分开发者工具版本不会自动创建触发器，此时按同名 cron 手动创建。

## 5. 权限验收

必须使用两个不同微信身份 A/B：

1. A 新建待办，A 能读取、完成和删除。
2. B 看不到 A 的待办。
3. 通过调试器让 B 尝试修改已知的 A 文档 id，操作必须被拒绝。
4. 小程序客户端尝试写 `articles`，操作必须被拒绝。
5. 云函数运行身份仍可写 `articles`。

把结果记录到 `docs/qa/p0-evidence.md`，只记录成功/失败和错误码，不记录完整 openid 或待办内容。

## 6. 常见问题

- 页面一直是本机模式：确认 `.env.local` 在构建前存在，重启 watch 进程。
- `permission denied`：检查集合权限和文档 `_openid`，不要在客户端伪造该字段。
- 阅读只有示例：检查 `articles` 是否有数据、客户端读权限和环境 ID。
- 同步函数网络失败：确认云函数运行环境能访问 `hacker-news.firebaseio.com`，失败时不要清空旧缓存。
