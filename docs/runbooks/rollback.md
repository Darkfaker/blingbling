# 回滚

## 前端故障

在微信公众平台或开发者工具中选上一可用版本重新设为体验版/提交发布。回滚前端时不要删除新版新增字段，旧客户端应能忽略它们。

## 云函数故障

1. 暂停 `sync-reading-every-6-hours` 触发器。
2. 回滚到上一云函数版本。
3. 手动执行并确认不会删除旧文章。
4. 恢复定时触发器。

## 权限故障

优先收紧：发现越权风险时立即把相关集合改为“仅管理端可读写”，宁可云工具暂时不可用。确认修复规则并完成 A/B 身份测试后再恢复。

## 数据故障

当前 v0.1 没有破坏性迁移。不要批量删除集合；先导出备份，再用 `schemaVersion` 编写可重复执行的修复脚本。文章同步使用固定 id，可以安全重跑覆盖。
