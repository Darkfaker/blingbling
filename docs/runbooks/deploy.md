# 体验版部署

1. 确认工作树中没有密钥、`.env.local`、真实用户数据或调试截图。
2. 执行：

   ```bash
   pnpm install --frozen-lockfile
   pnpm check
   ```

3. 按 `cloud-setup.md` 部署集合权限和 `syncReading`，先在测试环境手动同步一次。
4. 用微信开发者工具导入项目根目录，确认构建目录为 `dist/`。
5. 在开发版完成 `docs/qa/release-checklist.md`。
6. 上传小程序代码，版本号使用 `0.1.0`，备注写清数据模型或云函数变化。
7. 设为体验版，用 iOS 和 Android 各走一遍核心任务。
8. 确认云函数日志无持续错误后再提交审核或向熟人开放体验。

发布顺序始终是：向后兼容的数据库/权限 → 云函数 → 前端。不要先发布依赖新数据结构的前端。
