# ADR 0002：云身份与私有数据权限

- 状态：方案已接受，真实身份越权验证待 AppID
- 日期：2026-07-03

## 背景

待办和收藏需要用户隔离，但本地工具不应被登录流程阻塞，也不需要昵称、头像或账号密码。

## 候选

- 微信云开发原生身份与文档数据库安全规则
- 独立 CloudBase SDK 自定义登录
- 仅在前两者无法满足时使用 login 云函数

## 验证方法

用两个微信身份验证本人 CRUD、跨身份读取与修改拒绝、断网错误和权限规则的可复现部署。记录客户端查询条件以及是否真的需要 login 云函数。

## 证据

客户端已实现微信云开发原生身份路径，并在缺少云环境时自动使用本地仓储。安全规则与 A/B 身份验收步骤已写入 `cloud/rules/` 和 `docs/runbooks/cloud-setup.md`。当前环境没有用户 AppID，尚不能取得真实越权证据。

## 决定

采用微信云开发原生身份与文档数据库安全规则，不创建登录页、不读取昵称头像、不创建 login 云函数。

## 后果

`todos` 仅创建者可读写，`articles` 客户端只读。没有环境 ID 时待办明确显示本机模式，不伪装成已同步。正式开放前必须完成 runbook 中的双身份越权测试。
