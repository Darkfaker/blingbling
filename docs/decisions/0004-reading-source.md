# ADR 0004：阅读内容源或替代工具

- 状态：已接受
- 日期：2026-07-03

## 背景

阅读模块只有在来源合法、稳定且字段可形成幂等同步时才值得进入 v0.1。内容源不明确时先开发泛化抓取器会制造无效复杂度。

## 候选

- Hacker News 官方只读 Firebase API
- 移除阅读，替换为纯本地文本暂存工具

## 验证方法

检查许可与使用条款，解析 20 条数据，连续同步两次验证稳定 id，模拟来源失败并确认旧缓存可用。明确允许缓存的最小字段。

## 证据

采用 Hacker News 官方公开只读 API：`https://github.com/HackerNews/API`。同步函数只保存标题、原文链接、作者/分数摘要和时间，不保存或渲染第三方正文。固定文档 id `hn:<itemId>` 保证重复同步幂等。

## 决定

v0.1 采用 Hacker News 官方 API；云环境未配置时展示仓库内安全示例，不直接由客户端请求来源。

## 后果

云函数每 6 小时同步前 30 条。只接受 HTTP/HTTPS 链接，来源失败不清空旧集合。若 API 条款、稳定性或网络可达性变化，立即暂停定时器并保留旧缓存；不降级为抓取网页正文。
