# ADR 0003：二维码渲染与图片导出

- 状态：实现已接受，三端扫码待真机
- 日期：2026-07-03

## 背景

二维码必须在小程序真机可靠生成、保存并被常见扫码入口识别。依赖体积估算不能代替实测。

## 候选

- `qrcode-generator` 1.4.4：只生成矩阵，由项目自行绘制 Canvas。
- 服务器生成图片：因会上传用户内容而排除。

## 验证方法

测试 ASCII、中文、emoji、URL 和接近 500 字节内容；导出 1024×1024 白底图片；验证微信扫码、iOS 相机、Android 相机和相册权限各状态。

## 证据

已锁定 `qrcode-generator` 1.4.4，使用纠错级别 M、4 模块静区、640 逻辑像素画布与 1024×1024 PNG 导出。ASCII/中文/emoji 的 UTF-8 边界有自动化测试，生产构建成功。扫码与相册权限仍需 AppID 真机验证。

## 决定

采用 `qrcode-generator` 矩阵 + 微信 Canvas 本地绘制，不经过服务端。

## 后果

最大 500 UTF-8 字节，纠错级别 M，4 模块静区，导出 1024×1024 白底 PNG。保存权限失败时引导打开设置；生成失败时仍可复制原文。
