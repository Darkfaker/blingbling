# ADR 0001：小程序前端框架

- 状态：已接受（真机复核待 AppID）
- 日期：2026-07-03

## 背景

项目只发布微信小程序，但维护者熟悉 React/TypeScript。需要在 Taro 与微信原生 TypeScript 之间选择，不能以未来可能多端作为唯一依据。

## 候选

- Taro + React + TypeScript
- 微信原生 TypeScript

## 验证方法

用两个最小 spike 对比编译、导航、生命周期、测试接入、生产包大小、真机行为和非标准补丁成本。二维码与云开发核心能力也必须能在胜出方案中工作。

## 证据

Taro 4.2.0 + React 18.3.1 + TypeScript 5.8.3 已完成类型检查、27 个单元测试和微信生产构建。生产产物约 496KB，无框架源码补丁。沙箱内的 macOS Dynamic Store 探测会失败，正常用户环境构建成功；这属于执行环境限制，不进入项目代码。

## 决定

采用 Taro 4.2.0 + React 18.3.1 + TypeScript。React 19 暂不采用，减少小程序生态兼容变量。

## 后果

所有 Taro 包精确锁定为 4.2.0。微信生命周期、Canvas 和云权限仍须用真实 AppID 真机复核；若这些核心路径需要修改框架或长期维护非标准补丁，则回退微信原生 TypeScript。
